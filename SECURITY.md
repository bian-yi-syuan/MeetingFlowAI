# Security Policy

## 基本方針

MeetingFlowAI は、会議音声や文字起こしなど、プライバシー性の高い情報を扱うアプリです。  
そのため、現在のポートフォリオ版では、外部AI APIや独自バックエンドに依存しない Local-first 構成を重視しています。

録音データや会議内容は端末内保存を中心に扱い、将来的にクラウドAIと連携する場合も、APIキーをアプリ内に直接保存しない設計を前提としています。

---

## 現在の設計方針

- 会議音声は端末内に保存する設計
- 録音ファイルは App Sandbox 内で管理
- 外部AI APIキーを iOS アプリ内に直接保存しない
- GitHub に APIキー、署名鍵、Provisioning Profile などの秘密情報を含めない
- メール共有やファイル共有は、ユーザー操作を前提とする
- 会議音声や文字起こしデータの取り扱いに配慮する
- 将来的なクラウドAI連携時は、自前のバックエンド経由で安全に接続する

---

## クラウドAI連携を追加する場合の方針

将来的に OpenAI API、Google Speech-to-Text、Gemini API などの外部AIサービスと連携する場合、以下の方針を想定しています。

1. APIキーを iOS アプリ、plist、xcconfig、GitHub に含めない。
2. iOS アプリから直接AI APIを呼び出さず、自前のバックエンドを経由する。
3. 通信は HTTPS を使用する。
4. 送信する会議データの範囲をユーザーに明示する。
5. 会議音声、文字起こし、メールアドレス、認証情報をログに残さない。
6. 入力サイズ制限、タイムアウト、レート制限を設定する。
7. プライバシーポリシーと App Privacy の内容を実装内容に合わせて更新する。

---

## GitHub 公開時の注意

このリポジトリには、以下の情報を含めないようにしています。

- API Key
- Secret Token
- 署名鍵
- Provisioning Profile
- 個人情報を含む実際の会議データ
- 実在する会社や個人の会議音声
- 本番環境の設定ファイル

---

## 報告について

現在はポートフォリオ公開用のプロジェクトです。  
不具合や改善点がありましたら、Issue または Pull Request にてお知らせください。

なお、会議データ、音声ファイル、個人情報、APIキーなどの秘密情報は投稿しないようお願いいたします。
